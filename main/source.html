How This Website Was Made

Ever wonder what goes into building a website? Let me break it down for you in simple terms so you can build something like this to impress yourself and others!

üèóÔ∏è HTML (The Building Blocks)

Think of HTML like the skeleton and structure of a house. It tells the browser "put a heading here, a paragraph there, and a button over here." Every page you see (Home, About, Experience, etc.) is built with HTML. It's like writing instructions: "This is a title. This is some text. This is a link to another page."

üé® CSS (The Paint and Decorations)

CSS is like painting and decorating that house. It makes things look pretty! It decides the colors, fonts, spacing, and how things move when you hover over them. That cool blue color you see? That's CSS. The cards that pop out when you hover? Also CSS. It's what makes the website look professional instead of just plain black text on a white background.

‚ú® JavaScript (The Magic Tricks)

JavaScript makes the website interactive and fun! Remember how the welcome message types itself out? Or how the navigation cards slide in one by one? That's JavaScript doing its magic. It's like having a little helper that says "when someone scrolls down, make this appear" or "when they click here, do this cool animation." It brings the website to life!

üì± Responsive Design (Works Everywhere)

This website looks good whether you're on a phone, tablet, or computer. That's called "responsive design." It's like having clothes that automatically adjust to fit you perfectly no matter your size. The website rearranges itself to look great on any screen. On a phone, things stack vertically. On a big computer screen, they spread out nicely.

üéØ Custom Properties (Smart Shortcuts)

I used CSS custom properties (also called variables) to keep everything consistent. Imagine if you had to paint every wall in your house individually versus having one paint bucket you use everywhere. If you want to change the color, you just change the bucket! That's what custom properties do - they let me define colors, spacing, and fonts once, then use them everywhere.

üé¨ Animations and Transitions

All those smooth movements you see - cards sliding in, things fading, the hover effects - are carefully crafted animations. It's like choreographing a dance. Each element knows when to move, how fast to move, and where to go. This makes the website feel polished and professional instead of jumpy or boring.

ü§ñ AI Assistant (Kiro)

I used Kiro, an AI-powered code editor and assistant, to help build this website. Think of Kiro like having a really smart coding buddy who can write code, suggest improvements, and help solve problems. Instead of typing everything manually, I could describe what I wanted ("make the cards pop out on hover") and Kiro helped generate the code. It's like having a translator that turns my ideas into actual working code!

üíª Code Editor (Where the Magic Happens)

All the code was written in Kiro's built-in code editor. A code editor is like Microsoft Word, but for programmers. It helps you write code by color-coding different parts (so you can read it easier), catching mistakes, and organizing your files. Kiro's editor is special because it has AI built right in, so it can help you write better code faster.

üìÅ File Structure (Keeping Things Organized)

The website is organized into folders, just like organizing files on your computer. There's a folder for styles (CSS files), a folder for scripts (JavaScript files), and separate HTML files for each page. This keeps everything neat and easy to find. It's like having a filing cabinet where you know exactly where everything is.

üé® Design Principles

The website follows modern design principles: clean layout, consistent colors (that professional blue you see everywhere), good spacing so things aren't cramped, and clear navigation so you always know where you are. It's designed to be easy on the eyes and easy to use. Good design is invisible - you don't notice it, you just enjoy using the website.

‚ö° Performance (Fast and Smooth)

The website is built to load quickly and run smoothly. There are no heavy images or unnecessary code slowing things down. Every animation is optimized to be smooth. It's like having a sports car that's been tuned perfectly - it just works beautifully without any lag or stuttering.

üåê GitHub & GitHub Pages (Sharing with the World)

This website is hosted on GitHub Pages, which is like a free home for your website on the internet. GitHub is where developers store and share their code - think of it as Google Drive, but specifically for code. GitHub Pages takes your code and turns it into a live website that anyone can visit. It's completely free and automatically updates whenever you make changes to your code!

The source code for this entire website is publicly available on GitHub at github.com/idhdad171920/virtual-resume. This means you can see exactly how everything works, learn from it, or even use it as a starting point for your own website. That's the beauty of open source - we all learn from each other!

üöÄ Version Control with Git

Git is like a time machine for your code. Every time I make changes to the website, Git saves a snapshot. If something breaks, I can go back to when it was working. It also lets me try new features without worrying about messing up the main website. Think of it like having unlimited "undo" buttons and the ability to work on multiple versions at once. Every professional developer uses Git - it's an essential tool!

üéì The Bottom Line

Building a website is like building a house: you need a strong structure (HTML), beautiful decorations (CSS), and some cool features that make it special (JavaScript). With the help of AI tools like Kiro, anyone can learn to build websites. You don't need to be a genius - you just need curiosity and the willingness to learn!

Want to learn more? The best way to understand how websites work is to try building one yourself. Start small, experiment, and don't be afraid to break things - that's how we all learn! Check out the source code on GitHub to see exactly how this website was built.
